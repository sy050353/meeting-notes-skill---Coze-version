# 会议纪要处理 Skill

## 概述
标准化处理会议纪要的全流程技能。接收中文会议内容，生成符合规范的英文会议纪要，自动管理文件格式、状态流转与索引更新。

## 适用场景
- 用户口述/粘贴会议内容，需要生成英文纪要
- 查找历史会议纪要
- 维护会议纪要的格式与索引

## 核心流程

### 流程1：生成新会议纪要
```
用户提供会议内容
    ↓
读取 RULES.md 规则
    ↓
生成英文草稿
    ↓
质量自检（10项清单）
    ↓
用户确认
    ↓
写入当日文件（标记[待发送]）
    ↓
更新索引
```

### 流程2：查找历史纪要
```
用户查询某条/某类纪要
    ↓
通过索引快速定位（按日期/产品/参会方）
    ↓
读取对应文件返回详情
```

### 流程3：推送/发送纪要
```
用户请求发送待推送纪要
    ↓
扫描所有[待发送]的纪要
    ↓
逐条展示完整内容
    ↓
将状态从[待发送]改为[已发送]
    ↓
核对数量一致性
    ↓
更新索引
```

## 核心规范（详细见 references/RULES.md）

### 格式标准
- **标题格式**：产品名大写 | YOYO 月份缩写+日期 | w/ 角色/部门
- **语言**：全英文，完整句子
- **F/O**：有明确跟进事项写具体内容，无触发词统一写 `good for now`
- **状态标记**：`## [待发送]` / `## [已发送]` / `## [发送失败]`

### 质量自检清单
- [ ] 标题格式正确
- [ ] 日期格式正确（月份缩写+日期，不加年份）
- [ ] 全英文内容
- [ ] 要点是完整句子
- [ ] 术语正确（对照术语表）
- [ ] F/O 格式与内容正确
- [ ] 无 emoji
- [ ] 无具体时间（只保留日期）
- [ ] 视角正确（从 Yoyo 角度）
- [ ] 状态标记格式规范

## 快速开始

### 常见用法
1. **生成纪要**：「帮我写一份会议纪要，内容是...」
2. **发送纪要**：「现在发送待发送的会议纪要」
3. **查找纪要**：「找一下6月10号和RFQE相关的会议纪要」
4. **更新索引**：「更新会议纪要索引」

## 常用操作

### 1. 生成纪要草稿
读取 RULES.md → 分析用户提供的会议内容 → 生成符合格式的英文草稿 → 自检 → 提交用户确认

### 2. 写入纪要文件
用户确认后，写入对应日期的 .md 文件，标记为 `[待发送]`，然后更新索引

### 3. 发送待发送纪要
扫描所有文件中的 `[待发送]` 条目 → 逐条展示完整内容 → 更新状态为 `[已发送]` → 核对数量

### 4. 更新索引
运行 `scripts/generate_index.py --meeting-dir <纪要目录>` 重新生成所有索引文件

## 文件命名规则
- 每日纪要：`YYYY-MM-DD.md`（如 `2026-06-25.md`）
- 月度索引：`YYYY-MM.md`（如 `2026-06.md`）
- 总索引：`README.md`

## 注意事项
1. **规则唯一可信源**：所有规则以 `references/RULES.md` 为准，禁止凭记忆处理
2. **状态一致性**：发送后必须立即更新状态，避免重复发送
3. **数据准确性**：所有专业术语必须对照术语表核实
4. **备份机制**：批量修改前先备份原始文件
5. **路径配置**：脚本通过 `--meeting-dir` 参数指定纪要目录，不再硬编码路径
